The Mapbox GL style is an object that defines what data to draw, the order to draw it in, and how to style the data when drawing it. The JSON structure follows the renderer implementation very closely. It provides the basic building blocks out of which more complex styles can be built.

Key: required supports function values transitionable

{
"version": 4,

Stylesheet version number. Must be 4.

"sprite": "https://www.mapbox.com",

A base URL for retrieving the sprite image and metadata. The extensions `.png`, `.json` and scale factor `@2x.png` will be automatically appended.

"constants": { "@water": "#88aacc", "@land": "#ffffff" },

A constant that will be replaced verbatim in the referencing place. This can be anything, including objects and arrays. All variable names must be prefixed with an `@` symbol.

"sources": { "streets": { "type": "vector", "url": "mapbox://mapbox.mapbox-streets-v5", "tileSize": 512 } },

Render

Each layer can include a render object defining how data for that layer should be passed to the GPU. The render type determines how the layer data can be styled and should be one of fill, line, symbol, background, composite, raster.

Render: fill

type
Optional enum. One of fill.
fill-winding
Optional enum. One of non-zero, even-odd. Defaults to non-zero.

Render: line

type
Optional enum. One of line.
line-cap
Optional enum. One of butt, round, square. Defaults to butt.
The display of line endings.
line-join
Optional enum. One of bevel, round, miter. Defaults to miter.
The display of lines when joining.
line-miter-limit
Optional number. Defaults to 2.
Used to automatically convert miter joins to bevel joins for sharp angles.
line-round-limit
Optional number. Defaults to 1.
Used to automatically convert round joins to miter joins for shallow angles.

Values

Below is a list of values and an explanation of any expression that can be applied to properties in Mapbox GL.

Color

Mapbox GL accepts a variety of syntaxes for colors - HTML-style hex values, rgb, rgba, hsl, and hsla. It also supports the predefined HTML colors names, like yellow and blue.

{% highlight json %} { "line-color": "#ff0", "line-color": "#ffff00", "line-color": "rgb(255, 255, 0)", "line-color": "rgba(255, 255, 0, 1)", "line-color": "hsl(100, 50%, 50%)", "line-color": "hsla(100, 50%, 50%, 1)", "line-color": "yellow" } {% endhighlight %}

Especially of note is the support for hsl, which can be easier to reason about than rgb().

String

A string is basically just text. In the case of Mapbox GL, you're going to put it in quotes. Strings can be anything, though pay attention to the case of text-name - it actually will refer to features, which you refer to by putting them in curly braces, as seen in the example below.

{% highlight json %} { "text-name": "{{MY_FIELD}}" } {% endhighlight %}

Boolean

Boolean means yes or no, so it accepts the values true or false.

{% highlight json %} { "fill-enabled": true } {% endhighlight %}
